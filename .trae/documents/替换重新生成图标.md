## 更换网页标签页图标计划

### 问题分析
用户希望更换网页打开时显示的浏览器标签页图标。当前配置引用的是 `/icons/icon-192x192.svg`，但 icons 目录不存在，导致图标无法正常显示。

### 解决方案
1. **创建 icons 目录**：在项目根目录下创建 icons 文件夹
2. **生成图标文件**：创建一个美观的 SVG 图标文件，符合项目主题
3. **更新配置**：确保 index.html 和 vite.config.ts 中的图标引用正确
4. **测试效果**：验证图标在浏览器中正常显示

### 实施步骤
1. **创建 icons 目录**
   ```
   mkdir icons
   ```

2. **生成图标文件**
   - 创建 `icon-192x192.svg` 文件，使用蓝色主题，包含项目相关元素
   - 创建 `icon-512x512.svg` 文件，作为高分辨率版本
   - 确保图标符合 PWA 要求的尺寸和格式

3. **更新配置文件**
   - 确保 `index.html` 中的图标引用正确
   - 确保 `vite.config.ts` 中的 PWA 配置正确包含图标文件

4. **测试效果**
   - 重新启动开发服务器
   - 检查浏览器标签页图标是否正常显示
   - 验证 PWA 相关功能是否正常

### 图标设计建议
- 使用蓝色主题（#2563eb），与项目主题色一致
- 包含与津脉智坊相关的元素，如门、天津特色等
- 保持简洁现代的设计风格
- 确保在小尺寸下清晰可辨

### 预期效果
- 浏览器标签页显示美观的自定义图标
- PWA 应用显示正确的图标
- 整体视觉效果更加专业和统一
