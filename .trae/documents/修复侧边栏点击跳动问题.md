## 问题分析
用户反映的问题包括两个方面：
1. 侧边栏点击时会出现跳动现象
2. 当导航栏缩回（折叠）后，点击跳转其他页面时，导航栏会自动弹开（展开）

通过代码分析，主要原因有：

1. **主内容区域点击自动折叠**：在 `SidebarLayout.tsx` 第442行，主内容区域添加了点击事件，当点击时会自动折叠侧边栏，导致主内容区域突然变宽，产生跳动感。

2. **侧边栏折叠/展开动画不流畅**：当前侧边栏的折叠/展开没有平滑过渡效果，宽度变化较为突兀。

3. **折叠状态未持久化**：`collapsed` 状态仅存在于组件内存中，页面跳转时组件重新渲染，折叠状态丢失，恢复为默认的展开状态。

## 解决方案

### 1. 移除主内容区域的自动折叠功能
- 移除第442行的点击事件 `onClick={() => { if (!collapsed) setCollapsed(true) }}`
- 保留用户主动点击折叠按钮的交互方式，避免误触导致的意外折叠

### 2. 添加平滑的折叠/展开动画
- 为侧边栏添加 `transition: width 0.2s ease-in-out` CSS 属性
- 确保侧边栏宽度变化时平滑过渡，减少视觉跳动

### 3. 持久化折叠状态到 localStorage
- 修改 `collapsed` 状态的初始化逻辑，从 localStorage 读取保存的状态
- 添加 `useEffect` 钩子，在 `collapsed` 状态变化时将其保存到 localStorage
- 确保页面跳转时折叠状态保持不变

### 4. 优化导航项激活状态样式
- 确保导航项激活状态的样式变化不会影响布局
- 统一导航项的高度和内边距，避免激活时产生高度变化

## 实施步骤

1. 修改 `SidebarLayout.tsx` 文件，移除主内容区域的自动折叠点击事件
2. 为侧边栏添加平滑过渡动画
3. 实现折叠状态的持久化存储
4. 优化导航项激活状态样式
5. 测试侧边栏的交互效果，确保点击时不再跳动，页面跳转时保持折叠状态

## 预期效果
- 侧边栏点击时不再出现突然跳动
- 侧边栏折叠/展开动画平滑流畅
- 导航栏缩回后，点击跳转其他页面时不会自动弹开
- 导航项切换时视觉稳定
- 用户体验更加舒适自然
