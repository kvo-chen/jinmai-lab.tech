## 问题与目标
- 问题：模型选择卡片/方案卡片点击区域不明确、可选状态反馈弱，用户感知为“div 选择不了”；同时整体呈现层次略弱。
- 目标：增强选择可交互性（鼠标/键盘/可访问性）、提升选中反馈与视觉层次，并提供一键应用与连接测试。

## 变更范围
- ModelSelector（模型选择弹窗）：点击区域、键盘导航、选中高亮、快速应用与连接测试。
- Create（创作页方案选择）：方案卡片的选中交互与可访问性增强。

## 具体改动
### 1. ModelSelector 交互增强
- 将模型卡片由 `motion.div` 改为语义化 `button`：`type="button"`、`role="radio"`、`aria-checked`、`tabIndex`；保留动效与样式。
- 支持键盘选择：`onKeyDown` 响应 Enter/Space，左右方向键在模型列表循环焦点并切换选中。
- 明确选中态：加粗标题、边框加深、背景轻染、右上角勾选图标；focus 显示 `ring-2 ring-red-500`。
- 面板头部显示“当前模型：{name}”。
- 底部新增“立即应用”按钮：直接调用 `handleSave()` 并关闭弹窗。
- 新增“连接测试”按钮：调用 `llmService.generateResponse('ping')`（遵循现有超时/重试/流式），成功提示“连接正常”，异常提示并不更改配置。

### 2. Create 方案选择增强
- 方案卡片改为 `button`：`role="radio"`、`aria-checked`，扩大可点击区域（含图与信息栏），统一 hover/active/focus 表现。
- 键盘导航：方向键循环卡片、Enter 选中；选中后立即进入编辑步骤（维持现逻辑）。
- 在列表顶部保留“AI生成说明”，但折叠显示首行，支持“展开/收起”提高信息密度。

### 3. 可访问性与视觉
- 所有可选择项增加 `aria-label`、`aria-live`（选中提示），保证屏幕阅读器反馈。
- 使用 Tailwind `focus-visible` 与 `ring` 统一焦点样式；避免无效 `disabled` 传入到 `div`。

## 验证
- Settings 打开模型选择：鼠标点击与键盘切换均可正确切换模型与 API 配置区；点击“连接测试”成功/失败均给出 toast。
- Create 方案卡：鼠标与键盘均可选中，选中态视觉清晰；“AI生成说明”折叠可控。

## 风险与回退
- 所有更改均保持现有逻辑与样式体系，不引入新依赖；如出现异常，回退到 `div` 点击方案（保留函数与状态不变）。
