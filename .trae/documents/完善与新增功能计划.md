## 目标
- 提升“创作中心”可用性与闭环能力，新增草稿/预览/AI联动等高频功能。
- 增强 AI 对话、真实性评估与内容浏览体验，打通配置与路由入口。

## 技术栈概览
- 前端: React 18 + TypeScript + Vite 6 + Tailwind + Framer Motion。
- 路由: react-router-dom 7，私有/管理员路由守卫。
- AI 服务: `llmService` 支持 Kimi/DeepSeek，具备流式/重试，本地模拟与历史管理。
- 关键入口: `src/App.tsx:33-176` 路由；`src/pages/Create.tsx` 创作页；`src/components/LLMCommandPanel.tsx` 对话；`src/components/ModelSelector.tsx` 配置。

## 新增/完善功能
1. 创作工作台增强（Create）
- 自动保存草稿与进入页面自动恢复（localStorage），状态含 `prompt/selectedResult/currentStep`。
- 将“生成设计”改造为调用 `llmService.generateResponse`，支持流式输出并在右侧新增“AI生成说明”区实时展示。
- 增加“提示词模板”快捷选择（新增 `src/data/promptTemplates.ts`），辅助新手快速上手。
- 预览区支持基础 Markdown 文本预览（无需新库，使用简单渲染/`<pre>`），提升说明可读性。
- 预审弹窗接入 `llmService.diagnoseCreationIssues`，按 `prompt`/当前描述生成问题清单，与现有状态机融合。

2. AI 指令面板升级（LLMCommandPanel）
- 新增“导入/导出对话”JSON 按钮；导出当前 `conversation`；导入后合并并同步 `llmService` 历史。
- 在面板右上增加“流式开关”快捷按钮，直接调用 `llmService.updateConfig({ stream })`（与 ModelSelector 保持一致行为）。

3. 真实性评估可视化（Authenticity）
- 使用 `recharts` 增加相似度分布图/趋势图与问题条目列表，直观展示评估结果。
- 用户反馈持久化到 localStorage，便于后续分析与个性化建议。

4. Explore 列表分页与筛选
- 基于 `postService` 添加前端分页、标签筛选与排序；保留现有交互风格与主题适配。

5. 新增“设置”页面
- 新增 `src/pages/Settings.tsx`，路由 `/settings`（登录保护），集中呈现主题切换、API Key 输入与 ModelSelector 快捷入口。
